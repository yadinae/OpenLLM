# 系统架构

## 概述

OpenLLM 是一个统一的 AI 模型网关，支持多模型路由、智能评分、限流管理，以及 Agent 管理、沙盒执行、上下文优化和会话事件追踪。

## 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| Agent 识别中间件 | `agent_middleware.py` | 从请求头识别 Agent，应用配置 |
| Agent 注册表 | `agent_registry.py` | Agent 注册、API Key、用量追踪 |
| API 路由 | `router.py` | OpenAI 兼容端点 |
| 提示增强器 | `prompt_enhancer.py` | 代码思维 + 简洁模式 + 事件注入 |
| 代码思维 | `code_thinking.py` | 自动检测分析任务，引导用代码分析 |
| 简洁模式 | `terse_mode.py` | 输出压缩 65-75%，3 档强度 |
| 分发器 | `dispatcher.py` | 模型选择、故障转移 |
| 评分引擎 | `scorer.py` | 质量/速度/上下文/可靠性评分 |
| 模型注册表 | `registry.py` | 模型配置管理 |
| 协议适配器 | `adapters/` | OpenAI/Anthropic/REST/Ollama |
| 速率限制 | `limiter.py` | RPM/TPM 令牌桶限流 |
| 上下文管理 | `context.py` | 多轮对话上下文 |
| Token 优化 | `token_optimizer.py` | Token 级优化 |
| 复杂度评分 | `complexity_scorer.py` | 请求复杂度分析 |
| 沙盒系统 | `sandbox/` | 多语言代码执行 + FTS5 索引 |
| 会话追踪 | `session_tracker.py` | 事件提取 + BM25 搜索 |
| 管理面板 | `admin/index.html` | Web 仪表板 |

## 数据流

```
客户端请求 (X-API-Key)
    │
    ▼
Agent 识别中间件 → 识别 Agent、应用配置、隔离会话
    │
    ▼
提示增强器 → 代码思维 + 简洁模式 + 事件召回
    │
    ▼
复杂度评分 → 模型选择
    │
    ▼
分发器 → 执行请求 → 故障转移
    │
    ▼
记录用量 → 返回响应
```

详见 [配置指南](configuration.md)。

## 错误处理

| 错误 | 状态码 | 处理方式 |
|------|--------|----------|
| RateLimitError | 429 | 自动故障转移 |
| AdapterError | 500 | 日志 + 故障转移 |
| ConfigError | 400 | 返回错误 |
| TimeoutError | 504 | 重试 + 故障转移 |
| 无效 API Key | 401 | 拒绝 |
| Agent 已禁用 | 403 | 拒绝 |
